CSS

  • Enter your CSS notes here
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • CSS RULES example - div {color:blue; text-align:justify;}
  • Selector is at the start of the code (div) and defines the elements or attributes to which the rules or decloratons will apply
  • Declorations contains two important components: the CSS property we want to apply (color) and the value of the property (blue)
  • Do not Repeat Yourself (DRY) is a coding principle that encourages developers to reuse or share pieces of code inorder to reduce the number of lines of code that need to be written or maintained, DRY can greatly reduce the over all size and complesxity of the codebase
  • The value of the property can be chosen from grey to blue and a hex# can also be used

Git

  • Enter your Git notes here
  • git must start every Git statement
  • git status - checks what branch we are working on
  • git checkout -b branch-name - creates a new brance and switches to it
  • git revert - goes to previous version
  • git add -A - commits all files to working branch
  • git commit -m "description of change" - updates git repo with the working directory
  • git pull origin main - syncs the local branch wiht the remote branch in GitHub
  • git push origin "branch name" pushes changes made to GItHub to be merged
  • The defult branch in called main/master and all work should be done on a working/feature branch and once reviewed and tested can be merged to main branch
  • A feature branch can hold the code for a single feature where a development branch can hold the code for multiple feature branches waiting to be merged
