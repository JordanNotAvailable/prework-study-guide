CSS

  • Enter your CSS notes here
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • CSS RULES example - div {color:blue; text-align:justify;}
  • Selector is at the start of the code (div) and defines the elements or attributes to which the rules or decloratons will apply
  • Declorations contains two important components: the CSS property we want to apply (color) and the value of the property (blue)

Git

  • Enter your Git notes here
  • git must start every Git statement
  • git status - checks what branch we are working on
  • git checkout -b branch-name - creates a new brance and switches to it
  • git revert - goes to previous version
  • git add -A - commits all files to working branch
  • git commit -m "description of change" - updates git repo with the working directory
  • git pull origin main - syncs the local branch wiht the remote branch in GitHub
  • git push origin "branch name" pushes changes made to GItHub to be merged
  • The defult branch in called main/master and all work should be done on a working/feature branch and once reviewed and tested can be merged to main branch
  • A feature branch can hold the code for a single feature where a development branch can hold the code for multiple feature branches waiting to be merged

JavaScript

  • Enter your JavaScript notes here
  • JavaScript is case sensitive
  • To declare a variable you use the keyword var an after the keyword you give the variable a name
  • The name must be unique as it will referance all the code stored inside the variable
  • you then use an operater (=) to assign the specific data you want stored in the variable
  • E.G. var topics = "HTML", CSS, Git, JavaScript"; - topics is the variable name and all after the operator is the data stored in the variable
  • Each line of JavaScript ends in a semicolon (;)
  • All strings are surrounded by quotaion marks (")
  • The style.css file gets linked with the head element and the script.js gets linked with the body
  • Control flow is the order in which the computer executes code in a script
  • The order of code matters - you cant have console.log(topics) before you define what the varible topics is
  • (===) is the strictly equality operater and it checks to see if two values are equal and returns with a true result if they are equal and a false result if they are not
  • True and False are both Boolean values
  • An array is a single variable used to hold a group of data
  • Arrays are typically used to store data that is in some way related
  • E.G. var shape = ["triangle", "square", "pentagon", "circle"];
  • Arrays can be a custer of different data such as strings, numbers and Boolean values
  • E.G. var studentInfo = ["Lu", 54, true]; - this shows the students name, age and if they are enrolled, Lu is in quotation marks because it is a string, 54 is a number and true is a Boolean so they dont need quotation marks
  • Array Index allows you to access a singhle piece of data within an array
  • The first item in an array is 0 and continues sequentially in only whole numbers
  • Instead of writing out 1,000 lines of code to get all the numbers for each bit of data inside the array you can use a 'for' loop
  • E.G. for (var x=0; x shapes.length; x++) - var x=0 is declaring x is equal to 0 - x shapes.length because x is less then the numvber of shapes (4) the loop will keep running - x++ now everytime the loop repeats it will increase by one until the end of the array and then the loop will stop
  • Functions are a reusable block of code that preforms a specific task with the instructions given
  • To create a function you use the key word 'function' then followed by a unique name as you will use this name everytime you call the function
  • A function name is always followed by parenteses ()
